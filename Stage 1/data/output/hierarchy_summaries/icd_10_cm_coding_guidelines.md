# ICD-10-CM Coding Guidelines

These guidelines establish standardized protocols for healthcare data quality by defining how to code uncertain diagnoses, sequence principal conditions, and handle specific clinical scenarios like pressure injuries and underdosing. The rules prioritize accurate documentation interpretation, ensuring that coding reflects the provider's final diagnostic statements while adhering to specific constraints for unconfirmed or contrasting conditions.

**Key Features:**
- Non-Pressure Ulcer Coding: Assign unstageable codes only when eschar or grafts prevent clinical staging determination.
- Uncertain Diagnosis Coding: Code 'probable' or 'suspected' diagnoses as confirmed for inpatient admissions based on workup and treatment.
- Principal Diagnosis Sequencing: Sequence the condition that occasioned the admission after study, even if treatment was not performed.
- Underdosing Code Assignment: Use T36-T50 codes with characters '6' for underdosing, never as principal diagnoses.
- Diagnostic Statement Logic: Base code assignment on the provider's final diagnostic statement rather than clinical criteria alone.
- Sign and Symptom Coding: Report signs or symptoms as diagnoses when a definitive diagnosis is not established by the end of the encounter.
- Contrasting Diagnoses Sequencing: Code 'either/or' documented diagnoses as if confirmed and sequence based on admission circumstances.
- Deep Tissue Pressure Injury Coding: Assign L89.--6 codes exclusively for confirmed pressure-induced deep tissue damage.
- Syndrome Coding Index Guidance: Follow Alphabetic Index instructions for syndromes, coding manifestations if no unique code exists.
- Impending and Threatened Condition Coding: Code 'impending' or 'threatened' conditions as confirmed if they occurred, otherwise reference index subentries.
- G89 Pain Classification Guidelines: Assign F45.41 for pain exclusively related to psychological disorders and exclude category G89 codes in those instances.
- Postoperative Pain Coding Guidelines: Default to acute pain codes unless documentation specifies chronic status, excluding routine immediate postoperative pain.
- G89 Pain Code Sequencing Logic: Assign the specific G89 category code followed by the code identifying the specific site of pain for pain control encounters.
- Code G89.3 Neoplasm Pain Sequencing: Assign G89.3 for pain related to cancer or malignancy regardless of whether the pain is acute or chronic.
- Chronic Pain Classification and Coding: Classify chronic pain under subcategory G89.2 based on provider documentation without a specific time-frame definition.

**Notable Details:**
- Unstageable pressure ulcers (L89.--0) require clinical inability to determine stage due to eschar or grafts.
- Unspecified stage (L89.--9) is used when the stage is not documented in the medical record.
- Underdosing codes must never be assigned as principal or first-listed codes.
- Conflicting medical record documentation requires a provider query for clarification before coding.

- Chronic Pain Classification and Coding: Classify chronic pain under subcategory G89.2 based on provider documentation without a specific time-frame definition.

### Sepsis and Burn Coding Guidelines

These guidelines establish strict sequencing and assignment rules for coding complex infectious conditions like sepsis and MRSA alongside injuries such as burns. The core theme involves prioritizing the underlying systemic infection or highest severity condition first, while utilizing specific secondary codes for complications, carriers, or late effects.

**Key Features:**
- Sepsis Coding Sequencing Rules: Prioritize the underlying systemic infection as the principal diagnosis when severe sepsis is present on admission.
- Burn Injury Coding Guidelines: Sequence the code reflecting the highest degree of burn first when multiple burns are present.
- MRSA Infection and Carrier Coding: Assign the specific infection code with B95.62 for MRSA when no combination code exists for the causal organism.
- Septic Shock Coding Sequencing: Sequence the systemic infection code first, followed by R65.21 or T81.12, avoiding septic shock as a principal diagnosis.
- Sequela and Late Effect Coding Guidelines: Code the residual condition first followed by the sequela code to indicate the late effect.
- Severe Sepsis Coding Guidelines: Assign code A41.9 for unspecified organisms and require documentation of acute organ dysfunction before using R65.2.
- Burn Sequela Coding Guidelines: Use the 7th character 'S' to indicate late effects such as scars or joint contractures in burn encounters.
- Sequela Code Sequencing Logic: Treat the sequela code as secondary to the complication code based on the causal relationship.
- Sepsis and Localized Infection Coding Guidelines: Sequence the underlying systemic infection first when admission is for sepsis with a localized infection.
- Code Sequencing Logic for Encounter Notes: Assign two codes for 'code also' notes, with sequencing determined by the specific circumstances of the encounter.
- MRSA Combination Code Assignment: Use combination codes for conditions like sepsis or pneumonia that include the causal organism instead of B95.62.

**Notable Details:**
- Do not assign a code from subcategory Z16.11 for resistance to penicillins in the context of MRSA carrier coding.
- Query the provider if blood cultures are negative or inconclusive but clinical evidence of sepsis exists.
- The 7th character 'S' must be assigned to indicate sequela for late effects of burns or corrosions.
- Additional codes for acute organ dysfunctions must be assigned for septic shock.
- A sequela is defined as the residual effect produced after the acute phase of an illness or injury has terminated.
- Systemic infection code must be sequenced first for septic shock, followed by R65.21 or T81.12.
- Assign a minimum of 2 codes for severe sepsis: first the underlying infection, then R65.2.
- When a patient has both internal and external burns, the circumstances of admission govern the selection of the principal diagnosis.

- Complication Diagnosis Designation Logic: Designates a complication as the principal diagnosis only when the admission is specifically for its treatment.
- POA Status Determination Logic: Assigns 'Y', 'N', or 'U' based on whether a condition was present at the time of inpatient admission.
