# Clinical Coding & Compliance

These guidelines establish standardized protocols for ICD-10-CM coding to ensure accurate representation of diagnoses, complications, and patient histories through strict documentation requirements and specific logic for complex conditions.

**Key Features:**
- Burn and Corrosion Coding distinguishes injury types and classifies by depth and site while excluding depth differentiation for eyes and internal organs.
- Pain Code Differentiation assigns specific G89 codes for central and chronic pain syndromes only with explicit provider documentation.
- Combination Code Implementation utilizes single codes to capture definitive diagnoses alongside common symptoms or substance intent.
- Osteoporosis Coding separates systemic osteoporosis (M81) from fracture-specific codes (M80) based on the presence of pathologic fractures.
- Pregnancy Complication Coding applies specific O10 codes for hypertensive heart and kidney disease complicating pregnancy with required secondary codes.
- Factitious Disorder Coding differentiates between self-imposed (F68.1-) and other-imposed (F68.A) disorders.
- Substance Use Remission Coding requires clinical judgment and documentation to assign codes for disorders in remission.
- History Z Code Implementation uses personal history codes for past conditions with recurrence potential and family history codes for genetic risk.
- Retained Products of Conception Coding assigns specific O03.4 or O07.4 codes for subsequent encounters following spontaneous abortion or termination.
- Newborn Evaluation Coding utilizes Z05 codes for healthy newborns evaluated for suspected conditions later ruled out.
- Signs and Symptoms Coding omits additional codes for signs and symptoms routinely associated with a specific disease process.
- Fracture Coding Guidelines code fractures individually by site, defaulting to closed and displaced if unspecified.
- CKD Classification Logic interprets 'and' in titles as 'and' or 'or' and assigns codes based on CKD stages 1-5.
- Myocardial Infarction Coding uses category I22 for subsequent AMIs within 4 weeks of an initial event.
- Code Specificity Requirements mandate reporting diagnosis codes at the highest level of specificity available in the medical record.

**Notable Details:**
- Codes G89.0 and G89.4 require specific provider documentation and are distinct from general chronic pain terms.
- For categories T36-T65, combination codes eliminate the need for additional external cause codes.
- Category M81 is used for patients with osteoporosis who do not currently have a pathologic fracture.
- Secondary codes from heart or kidney disease chapters are required when assigning O10 codes.
- Codes for 'in remission' (F10-F19) are assigned only on the basis of provider documentation.
- Family history codes are used when a patient has a family member with a disease causing higher risk.
- O03.4 is for subsequent encounters following spontaneous abortion, while O07.4 is for elective termination.

### ICD-10-CM Coding Guidelines

These guidelines establish the standardized rules for assigning ICD-10-CM codes, ensuring accuracy through specific logic for conditions like fractures and myocardial infarctions while enforcing strict character requirements and note interpretations.

**Key Features:**
- Fracture Coding Guidelines: Code fractures individually by site, defaulting to closed and displaced if unspecified.
- CKD Classification Logic: Interpret 'and' in titles as 'and' or 'or' and assign codes based on CKD stages 1-5.
- Myocardial Infarction Coding: Use category I22 for subsequent AMIs within 4 weeks of an initial event.
- Code Specificity Requirements: Report diagnosis codes at the highest level of specificity available in the medical record.
- STEMI/NSTEMI Guidelines: Assign specific subcategories I21.0-I21.4 based on infarction type and evolution.
- Excludes1 Note Logic: Never use the excluded code simultaneously with the code above the note.
- Code Subdivision Rules: Use three-character codes only if not further subdivided and include the 7th character when applicable.
- Inclusion and Index Assignment: Use inclusion terms to identify conditions for which a specific code does not exist.
- Chapter 19 Character Requirements: Assign the 7th character based on active treatment status rather than encounter timing.
- NEC Index Logic: Use 'other' or 'other specified' codes when the medical record provides detail for which no specific code exists.
- Conventions and Instructional Notes: Apply general classification rules embedded in the Alphabetic Index and Tabular List.
