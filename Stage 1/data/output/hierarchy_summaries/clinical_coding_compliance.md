# Clinical Coding & Compliance

These guidelines establish a rigorous framework for ICD-10-CM coding that prioritizes maximum specificity, strict adherence to character limits, and precise interpretation of instructional notes to ensure accurate clinical representation. They define protocols for sequencing combination codes, handling exclusions, and assigning severity-based classifications while emphasizing the importance of provider documentation for conditions like substance use remission and factitious disorders. The rules ensure compliance by distinguishing between mutually exclusive conditions and requiring codes at the highest level of specificity available in the medical record.

**Key Features:**
- Burn and Corrosion Coding: Distinguishes thermal/electric/radiation burns from chemical corrosions and classifies by depth and site.
- Pain Code Differentiation: Assigns postoperative pain to Chapter 19 codes and reserves specific G89 codes for documented central or chronic pain syndromes.
- Combination Code Implementation: Prevents additional symptom codes when a combination code covers the diagnosis and avoids external cause codes for poisonings.
- Osteoporosis Coding: Separates site-specific fracture codes (M80) from general osteoporosis codes (M81) which do not include site.
- Factitious Disorder Coding: Assigns distinct codes for self-imposed (F68.1-) versus other-imposed (F68.A) factitious disorders.
- Substance Use Remission: Requires provider documentation to assign 'in remission' codes for mild substance use disorders.
- History Z Code Implementation: Uses personal history codes for past conditions and family history codes for genetic risk factors.
- Retained Products of Conception: Mandates specific codes (O03.4, O07.4) for subsequent encounters of incomplete abortion or failed termination.
- Newborn Evaluation Coding: Restricts Z05 codes to healthy newborns evaluated for suspected conditions not found, excluding symptomatic cases.
- Signs and Symptoms Guidelines: Excludes signs and symptoms routinely associated with a disease process from additional coding.
- Chapter Usage Restrictions: Prohibits obstetric codes on maternal records and maternal codes on newborn records, requiring current presence for Chapter 16.
- Z-Code Exposure Classification: Applies Z20 for communicable disease exposure and Z77 for hazardous health contact exposures.
- Transplant Complication Coding: Requires T86 codes only when complications affect the function of the transplanted organ.
- Fracture Coding Guidelines: Code fractures individually by site, defaulting to closed and displaced when status is unspecified.
- CKD Classification Logic: Interpret 'and' in titles as 'and' or 'or' and assign codes based on specific severity stages.

**Notable Details:**
- Codes from Category O10 must be paired with a secondary hypertension code to specify heart or kidney disease type.
- Category P36 specifically includes congenital sepsis.
- Codes for retained products of conception apply even if the patient was previously discharged with a complete abortion diagnosis.

### ICD-10-CM Coding Guidelines

These guidelines establish a rigorous framework for ICD-10-CM coding that prioritizes maximum specificity, strict adherence to character limits, and precise interpretation of instructional notes. The rules ensure accurate clinical representation by defining how to sequence combination codes, handle exclusions, and assign severity-based classifications for conditions like myocardial infarction and chronic kidney disease.

**Key Features:**
- Fracture Coding Guidelines: Code fractures individually by site, defaulting to closed and displaced when status is unspecified.
- CKD Classification Logic: Interpret 'and' in titles as 'and' or 'or' and assign codes based on specific severity stages.
- Myocardial Infarction Sequencing: Use category I22 for subsequent type 1 AMIs within 4 weeks and avoid I22 for type 2 or 4/5 AMIs.
- Code Specificity Requirements: Always report diagnosis codes at the highest level of specificity available in the medical record.
- Excludes1 Note Logic: Treat Excludes1 notes as mutually exclusive conditions that cannot be coded together unless unrelated.
- 7th Character Requirements: Assign the 7th character based on active treatment status rather than the timing of the provider encounter.
- Combination Code Sequencing: Do not assign additional codes for conditions included in a combination code unless a different cause is documented.
- Coding Syntax Definitions: Distinguish between brackets (synonyms/manifestations) and parentheses (nonessential modifiers) in the Tabular List.

**Notable Details:**

### ICD-10-CM Coding Guidelines

These guidelines establish standardized protocols for ICD-10-CM coding across diverse clinical scenarios, ensuring accurate diagnosis sequencing, appropriate code selection for symptoms and encounters, and correct reporting of present-on-admission status. The rules specifically address complex areas such as neoplasm classification, dementia severity, and the distinction between routine examinations and diagnostic encounters.

**Key Features:**
- Symptom and Sign Code Reporting: Permits symptom codes when a definitive diagnosis is not established by the provider.
- Etiology/Manifestation Sequencing: Requires underlying etiology to be sequenced first, prohibiting manifestation codes as principal diagnoses.
- Neoplasm Overlapping Classification: Mandates classification to subcategory .8 for malignant neoplasms overlapping contiguous sites.
- Unique Code Restriction: Limits reporting of each unique ICD-10-CM diagnosis code to once per encounter.
- Borderline Diagnosis Coding: Instructs coders to treat borderline diagnoses as confirmed unless a specific index entry dictates otherwise.
- Z02 Encounter Logic: Restricts Z codes for routine exams and requires diagnosis codes if a condition is discovered during the visit.
- Follow-Up Code Implementation: Sequences follow-up codes first when a condition is fully treated and no longer exists.
- POA Reporting Logic: Assigns 'Y', 'N', 'U', or leaves blank based on explicit documentation of condition presence at admission.
- Section 21 Factors and Screening: Requires coding factors influencing health status and screening codes when tests are performed.
- Dementia Classification: Classifies dementia based on etiology and severity, using unspecified codes if severity is undocumented.
- Outpatient Coding Guidelines: Applies Section I conventions to outpatient services while distinguishing them from inpatient reporting rules.
- Radiation and Chemotherapy Admissions: Assigns therapy codes (Z51.0, Z51.11, Z51.12) as principal diagnosis with complications sequenced after.

**Notable Details:**
- Ectopic tissue neoplasms must be coded to the site of origin mentioned in documentation.
- Bilateral conditions without distinct laterality codes are reported only once.
- Follow-up codes imply the condition has been fully treated and no longer exists.
- The 'U' POA assignment is used when documentation is unclear regarding presence on admission.
- For multiple conditions in a single code, 'Y' is assigned only if all were present on admission.
- Highest severity level code is assigned if dementia progresses during an inpatient stay.
- Terms 'encounter' and 'visit' are used interchangeably for outpatient service contacts.
- Complication codes for radiation/chemotherapy are sequenced after the principal therapy code.
