# Impending and Threatened Condition Coding
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** Impending and Threatened Condition Coding

**Key Behaviors:**
- Code conditions described as 'impending' or 'threatened' as confirmed diagnoses if they occurred.
- Reference the Alphabetic Index for subentry terms for 'impending' or 'threatened' if the condition did not occur.
- Assign the specific code if subterms for 'impending' or 'threatened' are listed in the index.
- Code the existing underlying condition(s) instead of the impending/threatened condition if subterms are not listed.

**Mandatory Coding Criteria:**
- Condition must be described at the time of discharge as 'impending' or 'threatened'.
- Verification of occurrence is required to determine if coding as a confirmed diagnosis is appropriate.
- Consultation of the Alphabetic Index is mandatory to check for specific subentry terms before assigning codes.
