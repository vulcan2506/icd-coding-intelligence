# Combination Code Assignment Logic
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** Combination Code Assignment Logic

**Key Behaviors:**
- A combination code is a single code used to classify two diagnoses, or a diagnosis with an associated secondary process (manifestation), or a diagnosis with an associated complication.
- Combination codes are identified by referring to subterm entries in the Alphabetic Index and by reading the inclusion and exclusion notes in the Tabular List.
- Assign only the combination code when that code fully identifies the diagnostic conditions involved or when the Alphabetic Index so directs.
- Multiple coding should not be used when the classification provides a combination code that clearly identifies all of the elements documented in the diagnosis.
- When the combination code lacks necessary specificity in describing the manifestation or complication, an additional code should be used as a secondary code.

**Requirements / Properties:**
- The combination code must fully identify the diagnostic conditions involved.
- The Alphabetic Index must direct the use of the combination code.
- Inclusion and exclusion notes in the Tabular List must be consulted to identify the code.
- All elements documented in the diagnosis must be clearly identified by the combination code to avoid multiple coding.
- An additional secondary code is required only when the combination code lacks necessary specificity for the manifestation or complication.

**Deprecated in this version:**
- Using multiple codes when a single combination code clearly identifies all documented elements.
- Assigning a combination code without verifying inclusion/exclusion notes in the Tabular List.
- Failing to consult the Alphabetic Index subterm entries for code identification.
- Omitting a secondary code when the combination code lacks specificity for a manifestation or complication.
- Classifying two diagnoses or a diagnosis with a secondary process using separate codes instead of a combination code when available.

**New in this version:**
- Explicit requirement to use subterm entries in the Alphabetic Index for identifying combination codes.
- Mandatory review of inclusion and exclusion notes in the Tabular List for code validation.
- Clarification that a combination code is a single code for two diagnoses, a diagnosis with a manifestation, or a diagnosis with a complication.
- Rule that multiple coding is prohibited if the combination code fully identifies all documented elements.
- Directive to add a secondary code only when the combination code lacks specificity for the manifestation or complication.
