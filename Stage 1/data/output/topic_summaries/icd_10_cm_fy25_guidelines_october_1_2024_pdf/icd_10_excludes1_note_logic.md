# ICD-10 Excludes1 Note Logic
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** ICD-10 Excludes1 Note Logic

**Key Behaviors:**
- An Excludes1 note indicates that the excluded code should never be used at the same time as the code above the note.
- Excludes1 is used when two conditions cannot occur together, such as a congenital form versus an acquired form of the same condition.
- If it is not clear whether two conditions involving an Excludes1 note are related or not, the coder must query the provider.
- If two conditions are unrelated to each other despite an Excludes1 note, both codes may be assigned together.
- Only one code should be assigned if the Excludes1 note represents mutually exclusive forms of the same condition.

**Requirements / Properties:**
- Documentation must confirm that the two conditions are unrelated to justify coding both together when an Excludes1 note exists.
- Provider query is mandatory when the relationship between conditions linked by an Excludes1 note is unclear.
- Coding must reflect that mutually exclusive conditions (e.g., congenital vs. acquired) cannot be reported simultaneously.

**Deprecated in this version:**
- Assigning both codes from an Excludes1 pair without clinical evidence of unrelatedness.
- Assuming an Excludes1 note always prohibits combined coding regardless of clinical context.
- Coding unrelated conditions together without verifying the lack of relationship when an Excludes1 note is present.

**New in this version:**
- Explicit requirement to query the provider when the relationship between conditions linked by an Excludes1 note is ambiguous.
- Recognition that unrelated conditions may be coded together even if an Excludes1 note exists, provided clinical justification is clear.
