# ICD-10 Excludes1 Note Logic
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** ICD-10 Excludes1 Note Logic

**Key Behaviors:**
- An Excludes1 note indicates that the excluded code should never be used at the same time as the code above the note.
- Excludes1 is used when two conditions cannot occur together, such as congenital versus acquired forms.
- If conditions involving an Excludes1 note are unrelated, both codes may be assigned together.
- If it is unclear whether two conditions involving an Excludes1 note are related, the provider must be queried.
- An Excludes1 note is defined as a 'pure excludes note' meaning 'NOT CODED HERE!'
- The strict exclusion applies specifically when two conditions represent mutually exclusive states of the same disease, such as congenital versus acquired forms.

**Mandatory Coding Criteria:**
- Verify if the two conditions are related or unrelated before assigning codes.
- Query the provider if the relationship between conditions is unclear.
- Ensure congenital and acquired forms of the same condition are not coded together.
