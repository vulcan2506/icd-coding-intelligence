# ICD-10-CM Code Specificity Requirements
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** ICD-10-CM Code Specificity Requirements

**Key Behaviors:**
- Diagnosis codes must be reported at their highest number of characters available.
- Codes must be assigned to the highest level of specificity documented in the medical record.
- A three-character code is valid only if it is not further subdivided by additional characters.
- Codes must be coded to the full number of characters required for that specific code.
- The seventh character must be included if applicable to the code.

**Requirements / Properties:**
- Codes must utilize the maximum number of characters available for the specific diagnosis category.
- Coding must reflect the highest level of detail documented in the medical record.
- Three-character codes are mandatory only when no further subdivision exists for the condition.
- Incomplete coding (missing required characters) renders the code invalid.
- The seventh character extension is mandatory when applicable to the code structure.

**Deprecated in this version:**
- Using three-character codes when fourth, fifth, sixth, or seventh characters are available and applicable.
- Reporting codes at a lower level of specificity than what is documented in the medical record.
- Omitting the seventh character when it is required for the specific diagnosis code.
- Treating a three-character code as valid when it has been subdivided by additional characters.
- Failing to utilize the full character length defined for a specific ICD-10-CM category.

**New in this version:**
- Mandatory inclusion of the seventh character for applicable codes to ensure validity.
- Explicit requirement to report codes at the highest level of specificity available.
- Definition of code validity based on the full character count required for the specific category.
