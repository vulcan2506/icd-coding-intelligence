# Excludes2 Note Logic Implementation
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** Excludes2 Note Logic Implementation

**Key Behaviors:**
- A type 2 Excludes note represents 'Not included here.'
- The excluded condition is not part of the condition represented by the code.
- A patient may have both the coded condition and the excluded condition at the same time.
- It is acceptable to use both the code and the excluded code together when appropriate.
- The Excludes2 note indicates that the exclusion is not mandatory.

**Requirements / Properties:**
- Documentation must support the presence of both conditions simultaneously.
- The conditions must be clinically distinct and not integral parts of each other.
- The coder must verify that the exclusion is not a mandatory 'Excludes1' type.
- Both codes must be assigned only if the patient meets the criteria for both.
- The assignment must align with the 'Not included here' definition of the Excludes2 note.

**Deprecated in this version:**
- Assigning only the primary code when both conditions are present.
- Treating Excludes2 notes as mandatory exclusions.
- Assuming the excluded condition is part of the primary condition.
- Failing to code the excluded condition when clinical evidence supports its presence.
- Using Excludes2 notes to prevent dual coding of unrelated conditions.

**New in this version:**
- Explicit permission to code both the primary and excluded conditions together.
- Clarification that Excludes2 notes indicate non-inclusion rather than mutual exclusivity.
- Requirement to assess clinical appropriateness for dual coding under Excludes2 notes.
- Updated interpretation allowing simultaneous assignment of excluded codes.
- Guidance to treat Excludes2 notes as informational rather than restrictive.
