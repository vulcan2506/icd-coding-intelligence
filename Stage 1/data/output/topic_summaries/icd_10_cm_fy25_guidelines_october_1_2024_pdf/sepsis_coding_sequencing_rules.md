# Sepsis Coding Sequencing Rules
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** Sepsis Coding Sequencing and R65.2 Assignment

**Key Behaviors:**
- Underlying systemic infection sequenced first if severe sepsis is present on admission and meets principal diagnosis definition.
- Code from subcategory R65.2 can never be assigned as a principal diagnosis.
- If severe sepsis develops during an encounter, underlying infection and R65.2 code are assigned as secondary diagnoses.
- For infection following a procedure, the procedure-related infection code is sequenced first, followed by sepsis code (T81.44) and R65.2 if severe.
- Only one code from category R65 should be assigned; do not additionally assign R65.1 for non-infectious origin cases.
- For sepsis following a postprocedural wound, assign a code from T81.41-T81.43, T81.49, O86.00-O86.03, or O86.09 to identify the specific site of infection before assigning the sepsis code.
- For infections following infusion, transfusion, therapeutic injection, or immunization, assign a code from subcategory T80.2 or T88.0- first, followed by the code for the specific infection.

**Mandatory Coding Criteria:**
- Severe sepsis must meet the definition of principal diagnosis to sequence the underlying infection first.
- Provider must be queried if documentation is unclear whether severe sepsis was present on admission.
- Code from subcategory R65.2 must be assigned with additional codes for any acute organ dysfunction.
- For non-infectious conditions leading to infection, the non-infectious condition code is sequenced first if it meets principal diagnosis definition.
- If both non-infectious condition and infection meet principal diagnosis definition, either may be assigned as principal diagnosis.
- If documentation is not clear whether severe sepsis was present on admission, the provider must be queried before finalizing codes.
- When assigning codes for severe sepsis, additional codes must be assigned for any acute organ dysfunction present.
