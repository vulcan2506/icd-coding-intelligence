# Unspecified Medical Coding Guidelines
*Source: ICD-10-CM FY25 Guidelines October 1, 2024.pdf*

**Feature:** Unspecified Medical Coding Guidelines

**Key Behaviors:**
- Codes titled 'unspecified' are for use when the information in the medical record is insufficient to assign a more specific code.
- For categories without a specific unspecified code, the 'other specified' code may represent both other and unspecified conditions.
- Coders must verify medical record sufficiency before assigning unspecified codes.
- Coders should utilize 'other specified' codes as a fallback when unspecified codes are unavailable for a condition category.
- Guidelines reference Section I.B.18 regarding the use of signs, symptoms, and unspecified codes.

**Requirements / Properties:**
- Medical record information must be insufficient to assign a more specific code for an unspecified code to be valid.
- The absence of a specific unspecified code in a category permits the use of the 'other specified' code to cover unspecified cases.
- Documentation must support the insufficiency of detail required to justify an unspecified code assignment.

**Deprecated in this version:**
- Assigning unspecified codes when sufficient information exists to determine a more specific diagnosis.
- Using unspecified codes as a default without verifying medical record insufficiency.
- Failing to consider 'other specified' codes when an unspecified code is not available for a condition.

**New in this version:**
- Explicit permission to use 'other specified' codes to represent both other and unspecified conditions in categories lacking an unspecified code.
- Mandatory reference to Section I.B.18 for guidance on signs, symptoms, and unspecified code usage.
