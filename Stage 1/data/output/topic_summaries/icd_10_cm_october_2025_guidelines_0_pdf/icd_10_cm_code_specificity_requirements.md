# ICD-10-CM Code Specificity Requirements
*Source: icd_10_cm_october_2025_guidelines_0.pdf*

**Feature:** ICD-10-CM Code Specificity Requirements

**Key Behaviors:**
- Codes must be reported at their highest number of characters available and highest level of specificity documented in the medical record.
- Only complete codes are permissible for reporting; categories or subcategories are not allowed.
- Any applicable 7th character extension is required for validity.
- A three-character code is valid only if it is not further subdivided by additional characters.
- Codes are invalid if they have not been coded to the full number of characters required for that specific code.

**Requirements / Properties:**
- Diagnosis codes must be used and reported at their highest level of specificity documented in the medical record.
- Any applicable 7th character extension is mandatory for the code to be valid.
- A three-character code is only permissible if it is not further subdivided by additional characters.
- Reporting must exclude categories or subcategories; only specific codes are allowed.
- Codes must be composed of 3, 4, 5, 6, or 7 characters as appropriate for the condition.

**Deprecated in this version:**
- Reporting categories or subcategories instead of specific codes.
- Using a three-character code when the code is further subdivided by additional characters.
- Submitting codes that have not been extended to the full number of characters required.
- Omitting the 7th character when it is applicable to the diagnosis.
- Failing to report codes at the highest level of specificity available in the documentation.

**New in this version:**
- Mandatory inclusion of the 7th character extension where applicable for code validity.
- Strict prohibition of using categories or subcategories for reporting purposes.
- Requirement to utilize the highest number of characters available for each diagnosis code.
- Explicit rule that a three-character code is invalid if further subdivision exists.
- Enforcement of full character extension to ensure code validity in claims processing.
