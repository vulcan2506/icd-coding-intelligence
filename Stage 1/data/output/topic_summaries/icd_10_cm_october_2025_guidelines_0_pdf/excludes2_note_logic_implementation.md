# Excludes2 Note Logic Implementation
*Source: icd_10_cm_october_2025_guidelines_0.pdf*

**Feature:** Excludes2 Note Logic Implementation

**Key Behaviors:**
- A type 2 Excludes note represents 'Not included here.'
- The excluded condition is not part of the condition represented by the code.
- A patient may have both the coded condition and the excluded condition at the same time.
- It is acceptable to use both the code and the excluded code together when appropriate.
- The Excludes2 note indicates that the excluded condition is distinct from the primary condition.

**Requirements / Properties:**
- Documentation must support the presence of both conditions simultaneously.
- The conditions must be clinically distinct and not integral parts of each other.
- The coder must verify that the patient meets criteria for both the primary code and the excluded code.
- Both codes must be assigned only if the clinical scenario justifies concurrent occurrence.
- The Excludes2 note must be interpreted as allowing, not prohibiting, dual coding.

**Deprecated in this version:**
- Assigning only the primary code when the excluded condition is also present.
- Treating Excludes2 notes as mandatory exclusions preventing dual coding.
- Assuming the excluded condition is a component of the primary condition.
- Failing to code the excluded condition when clinical evidence supports its presence.
- Using Excludes2 notes to justify coding only one condition when both are documented.

**New in this version:**
- Explicit permission to code both the primary condition and the excluded condition concurrently.
- Clarification that Excludes2 notes indicate non-inclusion rather than mutual exclusivity.
- Requirement to assess clinical appropriateness for dual coding when an Excludes2 note is present.
- Updated guidance emphasizing that Excludes2 notes do not preclude simultaneous diagnosis.
- Mandatory verification of distinct clinical entities when applying Excludes2 logic.
