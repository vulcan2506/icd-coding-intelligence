# Septic Shock Coding Sequencing
*Source: icd_10_cm_october_2025_guidelines_0.pdf*

**Feature:** Septic Shock Coding Sequencing

**Key Behaviors:**
- Systemic infection code must be sequenced first for septic shock cases
- Code R65.21 or T81.12 follows the infection code based on etiology
- Additional codes for acute organ dysfunctions must be assigned
- Septic shock code cannot be assigned as a principal diagnosis
- Postprocedural septic shock requires specific sequencing without R65.21

**Requirements / Properties:**
- Documentation must confirm circulatory failure associated with severe sepsis
- Systemic infection etiology must be clearly identified
- Acute organ dysfunction must be documented for additional coding
- Postprocedural infection must be linked to the septic shock event
- Principal diagnosis must reflect the underlying infection, not the shock state

**Deprecated in this version:**
- Assigning septic shock code as principal diagnosis
- Using R65.21 for postprocedural septic shock cases
- Omitting codes for acute organ dysfunctions in septic shock
- Sequencing infection code after the septic shock code
- Failing to distinguish between severe sepsis and postprocedural septic shock

**New in this version:**
- Explicit sequencing rule placing infection code first
- Mandatory inclusion of acute organ dysfunction codes
- Prohibition of R65.21 in postprocedural septic shock scenarios
- Clarification that septic shock is a type of acute organ dysfunction
- Requirement to query provider if postprocedural infection is unclear
