# Sepsis Coding Sequencing Rules
*Source: icd_10_cm_october_2025_guidelines_0.pdf*

**Feature:** Sepsis Coding Sequencing and R65.2 Assignment

**Key Behaviors:**
- Underlying systemic infection is sequenced first if severe sepsis is present on admission and meets principal diagnosis criteria.
- Code from subcategory R65.2 can never be assigned as a principal diagnosis.
- If severe sepsis develops during an encounter, the underlying infection and R65.2 code are assigned as secondary diagnoses.
- For infection following a procedure, the procedure-related infection code is sequenced first, followed by sepsis code T81.44 or O86.04, then R65.2 if severe sepsis is present.
- Only one code from category R65 should be assigned; do not additionally assign R65.1 for non-infectious conditions leading to sepsis.
- For sepsis following infusion, transfusion, therapeutic injection, or immunization, code the specific complication (T80.2 or T88.0-) first, followed by the specific infection code.
- When a noninfectious condition leads to an infection resulting in sepsis, sequence the noninfectious condition first if it meets principal diagnosis criteria, followed by the infection code.

**Requirements / Properties:**
- Severe sepsis must be documented as present on admission to be sequenced with the underlying infection as principal diagnosis.
- Documentation must be clear regarding whether severe sepsis was present on admission; if unclear, the provider must be queried.
- The underlying systemic infection must be identified and coded when severe sepsis is present.
- Additional codes must be assigned for any acute organ dysfunction associated with severe sepsis.
- For infection-associated hemolytic-uremic syndrome, code D59.31 is the principal diagnosis, with infection and severe sepsis as secondary.
- An additional code must be used to identify the infectious agent when coding sepsis following a procedure or infusion/transfusion.
- Documentation must clearly establish whether severe sepsis was present on admission or developed during the encounter.

**Deprecated in this version:**
- Assigning a code from subcategory R65.2 as a principal diagnosis.
- Assigning both R65.1 and R65.2 when a non-infectious condition leads to sepsis.
- Sequencing the sepsis code before the underlying infection or procedure-related infection code when applicable.
- Failing to query the provider when documentation is unclear about the presence of severe sepsis on admission.
- Omitting codes for acute organ dysfunction when severe sepsis is present.
- Assigning a code from subcategory R65.1 (SIRS of non-infectious origin) when a noninfectious condition leads to an infection resulting in severe sepsis.
- Sequencing the sepsis code before the underlying infection or the procedure-related infection code when applicable.

**New in this version:**
- Explicit rule that R65.2 cannot be a principal diagnosis.
- Mandatory sequencing of procedure-related infection codes (T81.41-T81.43, T81.49, O86.00-O86.03) before sepsis codes.
- Requirement to assign D59.31 as principal diagnosis for infection-associated hemolytic-uremic syndrome.
- Clarification that R65.1 is not necessary when a non-infectious condition leads to sepsis.
- Guidance to query providers if severe sepsis status on admission is unclear.
