# Combination Code Assignment Logic
*Source: icd_10_cm_october_2025_guidelines_0.pdf*

**Feature:** Combination Code Assignment Logic

**Key Behaviors:**
- A single code classifies two diagnoses or a diagnosis with an associated secondary process/complication.
- Codes are identified via subterm entries in the Alphabetic Index and inclusion/exclusion notes in the Tabular List.
- Assign only the combination code when it fully identifies all diagnostic conditions or when the Alphabetic Index directs it.
- Multiple coding is prohibited when a combination code clearly identifies all documented elements.
- An additional secondary code is required if the combination code lacks necessary specificity for the manifestation or complication.

**Mandatory Coding Criteria:**
- The combination code must fully identify all diagnostic conditions involved.
- The Alphabetic Index must direct the use of the combination code.
- Inclusion and exclusion notes in the Tabular List must be consulted to identify combination codes.
- Specificity must be sufficient to describe the manifestation or complication without additional codes.
- All elements documented in the diagnosis must be clearly identified by the single combination code.
