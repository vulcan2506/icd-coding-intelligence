# Uncertain Diagnosis Coding Guidelines
*Source: icd_10_cm_october_2025_guidelines_0.pdf*

**Feature:** Uncertain Diagnosis Coding Guidelines

**Key Behaviors:**
- Code uncertain diagnoses (e.g., 'probable', 'suspected') as if they existed or were established for inpatient admissions.
- Basis for coding uncertain diagnoses includes diagnostic workup, arrangements for further workup, and initial therapeutic approach.
- Do not code diagnoses documented with uncertainty terms for outpatient encounters; instead, code to the highest degree of certainty (symptoms, signs, abnormal results).
- Apply inpatient coding rules only to short-term, acute, long-term care, and psychiatric hospitals.
- Differentiate coding practices between inpatient settings (code as established) and outpatient settings (code as symptoms/signs).

**Requirements / Properties:**
- Diagnosis must be qualified by terms like 'probable', 'suspected', 'likely', 'questionable', 'possible', 'still to be ruled out', 'compatible with', or 'consistent with'.
- Setting must be an inpatient admission to a short-term, acute, long-term care, or psychiatric hospital.
- Diagnostic workup, arrangements for further workup, or initial therapeutic approach must correspond to the established diagnosis.
- For outpatient visits, code must reflect the highest degree of certainty (symptoms, signs, abnormal test results) rather than the uncertain diagnosis itself.
- Documentation must explicitly indicate uncertainty to trigger the specific coding rule for inpatient settings.

**Deprecated in this version:**
- Coding uncertain diagnoses as if they were established for outpatient encounters.
- Applying inpatient 'code as established' rules to outpatient visits.
- Using uncertain diagnosis codes (e.g., R codes for symptoms) as the primary reason for visit in inpatient settings when the uncertain diagnosis is the focus of care.
- Ignoring the setting (inpatient vs. outpatient) when determining whether to code an uncertain diagnosis as established.
- Coding 'working diagnosis' as a confirmed condition in outpatient settings.

**New in this version:**
- Explicit instruction to code uncertain diagnoses as established for inpatient admissions based on therapeutic approach and workup.
- Clarification that outpatient coding for uncertain diagnoses must default to symptoms, signs, or abnormal test results.
- Specific list of uncertainty terms ('probable', 'suspected', 'likely', 'questionable', 'possible', 'still to be ruled out', 'compatible with', 'consistent with', 'working diagnosis').
- Mandatory distinction between inpatient and outpatient coding practices for uncertain diagnoses.
- Requirement to validate the basis of coding (workup, observation, therapy) for uncertain inpatient diagnoses.
